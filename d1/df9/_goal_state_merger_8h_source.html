34  /* Computes a submodel of the specified model that only considers the states given by maybeStates as well as
35  * * one target state to which all transitions to a state selected by targetStates are redirected and
36  * * one sink state to which all transitions to a state selected by sinkStates are redirected.
37  *
38  * If a choiceFilter is given, choices on maybestates that are not selected by the filter will be removed.
39  *
40  * Notes:
41  * * the target (or sink) state is not created, if it is not reachable
42  * * the target (or sink) state will get a label iff it is reachable and at least one of the given targetStates (sinkStates) have that label.
43  * * Only the selected reward models will be kept. The target and sink states will not get any reward.
44  * * Choices that lead from a maybeState to a ~(target | sink) state will be removed. An exception is thrown if this leads to deadlocks.
45  * * It is assumed that maybeStates, targetStates, and sinkStates are pairwise disjoint. Otherwise an exception is thrown.
46  * * The order of the maybeStates will not be affected (i.e. s_1 < s_2 in the input model implies s'_1 < s'_2 in the output model).
47  */
